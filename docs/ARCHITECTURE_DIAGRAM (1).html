Track 01 — Data & AI Challenge · Redrob Hackathon 2025

Intelligent Candidate Discovery

Complete System Architecture — CodeCrafter · Team Leader: Varsha Katiyar

00 · Data Input
↓
candidates.jsonl
Full candidate pool — profiles, skills, career history, education, certifications, redrob_signals (23 fields)
job_description.txt
Raw JD text — extracted from provided job_description.docx
01 · Understanding
↓
JD Parser
Extracts: required skills, nice-to-have, seniority level, experience range, work mode, domain intent
Profile Normalizer
Skill taxonomy normalization (JS→JavaScript), deduplication, candidate text blob construction
Skill Weighting new
Proficiency × endorsements × duration × assessment score → weighted skill vector per candidate
02 · Embedding
↓
TF-IDF Vectorizer
50k features, bigrams (1,2), sublinear TF. Converts candidate text blobs to sparse term matrix
TruncatedSVD (LSA)
Reduces to 256-dim dense vectors. Captures latent semantic topics — "neural nets" ≈ "ML Engineer"
Embedding Cache new
Fitted model + vectors saved to disk. Subsequent runs skip re-embedding entirely — instant reload
03 · Retrieval
↓
LSA Cosine Search
Dot product on L2-normalized vectors = cosine similarity. Top-1000 candidates retrieved by semantic match
BM25 Keyword Search
BM25Okapi over tokenized profiles. Catches exact skill terms semantic may miss (e.g. "FAISS", "NDCG")
RRF Fusion new
score = 0.6/(60+sem_rank) + 0.4/(60+bm25_rank). Combines both signals without hyperparameter tuning
04 · Scoring
↓
Semantic Score
Cosine similarity → 0–100. Meaning-level match between JD and candidate
Default weight: 40%
Profile Score
Skills (50%) + Experience (25%) + Education (15%) + Title (10%) + Nice-to-have bonus (up to +15)
Default weight: 35%
Behavioral Score
All 23 Redrob signals: Recency (30%) + Engagement (25%) + Reliability (25%) + Intent (20%)
Default weight: 25%
Dynamic Weight Tuning new
Senior roles → profile weight ↑. Fresher roles → behavioral weight ↑. 6 seniority levels configured
Cold-Start Multiplier new
Sparse behavioral profiles get 0.80–0.90× multiplier instead of score=0. Fair treatment, not exclusion
Final Score = (w_sem × Semantic + w_prof × Profile + w_beh × Behavioral) / 100
            × Cold-Start Multiplier (0.80 – 1.00)
            → clipped to [0.0, 1.0], rounded to 4 decimal places
05 · Ranking
↓
Ranking Engine
Sort by (-score, candidate_id). Pad to exactly 100. Enforce monotonically non-increasing scores
Explainability new
"Senior ML Eng 6.0 yrs; 3 skills matched; resp 0.85; active 2d ago; github 72; open to work"
Bias Audit new
Score distribution by experience bucket (0-2, 2-5, 5-9, 9+ yrs) and by country. Saved to JSON
06 · Output
↓
submission.csv
Exactly 100 rows. Columns: candidate_id, rank (1–100), score (0.0–1.0), reasoning
submission_detailed.json
100 rows with sub-scores: semantic, profile, behavioral. For analysis and debugging
bias_audit.json
Fairness report: count, avg, min, max score per experience bucket and country

Streamlit Live Demo bonus
streamlit run demo/app.py — paste any JD, see live ranked candidates instantly. Download CSV from UI
Speed Benchmark bonus
Displays "N candidates ranked in X.XX seconds" — quantified performance shown to judges in real time
